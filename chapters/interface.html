Interface In Java

Earlier we had discuss about Abstract Class that is used to achieve partial abstraction i.e; it can consist of both abstract as well as concerete method. In this section we are going to discuss about interface which is used to achieve 100% abstraction.

Brief Introduction about Interface In Java

Interface is a Java type which looks same as the class. It has methods and variable just like class but the method declared are of abstract in nature(only method signature and no body). Also the variable declared in an interface are public,static & final by default and interface cannot be initialized.

The interface variable has to be initialize at the time of declaration.

Before discussing about abstract class lets have a brief introduction about Abstraction.

Advantages of Interface

1). Using Interface we can achieve 100% abstraction which means we can achieve generalization across the sub-class. By generalization we can segregate the functionality with the implementation part of the code. Interface provides pure abstraction as compared to abstraction class.

2). Interface is used to acheive run-time polymorphism.

3). Java programming language doesn't support multiple inheritance, using interface we can achieve multiple inheritance.

Program 1) Below shows the Declaration of interface

Declaration

interface MyInterface{
/* All the methods are public abstract by default
 * Note down that these methods are not having body
 */
 /* All the variable are public,static & final by default*/
 int k = 0;
 public void method1();
 public void method2();
 }

In the above example we can see that there is variable and methods in the interface. The variable type is public,static & final and method is public and abstract by default.

Interface Implementation

A class can inherit from an interface using "implements" keyword. Whenever a class implements an interface it must define all the inherited abstract method otherwise class should be declares as abstract.

NB: A Class extends from another Class. A Class implements from an Interface. Interface extends from another interface.

Implementing Interface

interface Demo1{
void test1();
 void test2();
 }

class Demo2 implements Demo1 {
public void test1(){
 System.out.println("test1() implemented in Sample");
 }
 public void test2(){
 System.out.println("test2() implemented in Sample");
 }
 }

public class Sample{
public static void main(String[] args){
 System.out.println("Main Method Started");
 Demo2 ref = new Demo2();
 ref.test1();
 ref.test2();
 System.out.println("Main Method Ended");
 }
 }

In the above example we can see that Class Demo2 inherits from the interface Demo1. Since it implements the interface it had deined all the abstract method of the Abstract Class Demo1.

Multiple Inheritance In Java

Multiple Inheritance is not suported in Java. However we can achieve multiple inheritance using interface.

Why Java doesn't support Multiple Inheritance?

C++ and few other progamming languages support multiple inheritance while java doesn't support it. This is been done to remove ambiguity, beacuase multiple inheritance can cause ambiguity in few scenario. One of the most common scenar

Understanding the real scenario of the Abstract Class

In the example shown below we will see that there is an Abstract Class Shape with an Abstract Method draw(). The class Circle and Rectangle extends from the Abstract Class and implements it abstract method according to it own specification. Then we can reference the draw() method of the Circle and Rectangle class using Shape class reference variable. Thus we are segregating the functionalities with the implementation part of the code.

Sample Code

abstract class Shape{
abstract void draw();
 }

class Circle extends Shape{
void draw(){
 System.out.println("Drawing Circle");
 }
 }

class Rectangle extends Shape{
void draw(){
 System.out.println("Drawing Rectangle");
 }
 }

public class Sample {
public static void main(String[] args){
 System.out.println("Main Method Started");
 Shape ref1 = new Circle(); //implicit upcasting
 Shape ref2 = new Rectangle();//implicit upcasting
 ref1.draw(); //Calls the draw method of the Circle class
 ref2.draw(); //Calls the draw method of the Rectangle class
 System.out.println("Main Method Ended");
 }
 }

The output of the above program is as follows:

Output

Main Method Started
Drawing Circle
Drawing Rectangle
Main Method Ended

Extending Abstract Class

1). A class can inherit the abstract class. When class inherits from the abstract class, the class must be declared as abstract or class must provide implementaion of the inherited abstract method of the parent class.

2). It means that, whenever a class tries to build a "IS-A" relationship with an Abstract Class then a contract exist between Abstract Class and Subclass.

3). The contract specifies that the SubClass should be define all the inherited Abstract Method otherwise the Sub-Class should be declared as Abstract Class.

The example below shows how to inherit the abstract class.

Sample Code

abstract class Sample1{
abstract void test();
 }

abstract class Sample2 extends Sample1{
//The Class inheriting is abstract
 }

class Sample3 extends Sample2{
//Providing implementation of the abstract method test()
 void test(){
 System.out.println("Providing Implementation of the test()");
 }
 }

public class Test {
public static void main(String[] args){
 System.out.println("Main Method Started");
 Sample3 ref1 = new Sample3();
 ref1.test();
 System.out.println("Main Method Ended");
 }

 }

Output

Main Method Started
Providing Implementation of the test()
Main Method Ended

In the example shown above we can see that Class Sample2 inherits from the Abstract Class Sample1. Since it doesn't provide the implementation of the inherited Abstract Method it is defined as Abstract Class. On the other hand Class Sample3 inherits from the Abstract Class Sample2 but it is not defined as abstract since it provides implementation of the abstract method test().

Example below shows another example of inheritance in Abstract Class

Sample Code

abstract class SampleClass1{
abstract void test1();
 abstract void test2();
 }

abstract class SampleClass2 extends SampleClass1{
void test1(){
 System.out.println("Running test1() method in SampleClass2 class");
 }
 }

class SampleClass3 extends SampleClass2{
void test2(){
 System.out.println("Running test2() method in SampleClass3 class");
 }
 }

public class Sample1 {
public static void main(String[] args){
 System.out.println("Main Method Started");
 SampleClass1 ref = new SampleClass3();
 ref.test1();
 ref.test2();
 System.out.println("Main Method Ended");
 }
 }

In the above example the Class SampleClass2 inherits from Class SampleClass1 and it won't implement all the abstract method of the Class SampleClass1. Thus this class is declared as Abstract. But the Class SampleClass3 inherits Class SampleClass2 and it implements Class SampleClass2 abstract method test2(). Since all the abstract method of SampleClass2 is been implemented it is not declared as Abstract.
