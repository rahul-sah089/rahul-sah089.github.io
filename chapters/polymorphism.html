Polymorphism In Java

Object showing different behaviour at the different stages of it's lifecycle is knows as Polymorphism. It is the concept by which we can perform a single action in different ways.

There are two types of polymorphism:

1).Compile Time Polymorphism

1).Run Time Polymorphism

We can perform polymorphism in java using method overloading and method overriding.

In compile time polymorphism the method declaration is binded to the method declaration at the time of compilation. This type of binding is static binding which cannot be changed later Ex: static method,final method and private method. Here we will discuss about run time polymorphism is detail.

Run Time Polymorphism

In runtime polymorphism the method declaration is binded to it's definition by JVM at the time of execution. It is also know as Dynamic Method Dispatch in which a call to overriden method is resolved at the runtime rather than compile-time.

In this process, an overriden method is called through the reference variable of a superclass. The determination of the method behaviour is based on the object being referred by the reference variable.

To achieve run time polymorphism we need to have the following:

1) Interface

2) Method Overriding

3) Type Casting

Example of Java Polymorphism

In the example shown below we are having two class. Class Car and Car Bugati. Class Car is the parent class of the Car Bugati. Both the class have the method called run() but this method is been overriden in the base Class Bugati. We are now calling the run method with the reference variable of the Parent class.Since the reference variable now point to the object of it's base class, subclass method is invoked at the runtme.

Example of run time polymorphism

void run(){
 System.out.println("running");
 }
}
class Bugati extends Car{

Real example of Java Runtime Polymorphism


Consider a scenario, Car is the parent class that provides functionality to get the Top Speed of the Car. But, Top Speed varies from car to car so it's implementation will be overrided in the overriding method in the base class as shown below.

The source code for this implementation is shown below:

Source Code

class Car{
int cc;
 public double getTopSpeed(){
 return 0.0;
}
}

class Bugati extends Car{
public double getTopSpeed(){
 return 408.84;
 }
 }

class Ferrari extends Car{
public double getTopSpeed(){
 return 363;
 }
 }

class Pagani extends Car{
public double getTopSpeed(){
 return 335;
 }
 }

class Porshe extends Car{
public double getTopSpeed(){
 return 310;
 }
 }

public class TestVehicles {
public static void main(String[] args){
 System.out.println("Main Method Started");
 Car obj1 = new Bugati();
 Car obj2 = new Ferrari();
 Car obj3 = new Pagani();
 Car obj4 = new Porshe();
 System.out.println("Top speed of Bugati::"+obj1.getTopSpeed());
 System.out.println("Top speed of Ferrari::"+obj2.getTopSpeed());
 System.out.println("Top speed of Pagani::"+obj3.getTopSpeed());
 System.out.println("Top speed of Porshe::"+obj4.getTopSpeed());
 System.out.println("Main Method Ended");
 }
 }

Output of the above program in given below::

Top speed of Ferrari::363.0
Top speed of Pagani::335.0
Top speed of Porshe::310.0
Main Method Ended

Java Runtime Polymorphism with Multilevel Inheritance

Let's see the simple example of Runtime Polymorphism with multilevel inheritance.

class Animal{
void eat(){
 System.out.println("eating");
 }
}

class Dog extends Animal{
void eat(){
 System.out.println("eating fruits");
 }
}

class BabyDog extends Dog{
void eat(){
 System.out.println("drinking milk");
 }

public static void main(String args[]){
Animal a1,a2,a3;
a1=new Animal();
a2=new Dog();
a3=new BabyDog();

a1.eat();
a2.eat();
a3.eat();
}
}

Output

Output: eating
eating fruits
drinking Milk
