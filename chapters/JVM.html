JVM Introduction

A Java Virtual Machine(JVM), an implementation of the Java Virtual Machine Specification, intercept compiled Java binary code(byte code) for a particular machine, processor(or "hardware Platform") so that is can perform Java program's instruction

As show is the diagram, the java compiler takes the Java instruction, convert them into byte code(.class). This byte code is "platform independent". Now this byte code can run easily on the various machine and operating system

To know more about virtual Machine. Click Here

There are two types of virtual machines

1). Hardware Based Virtual Machine

2). Software Based Virtual Machine

Hardware Based Virtual Machine

Hardware Based Virtual Machine is also System Based Virtual Machine.Hardware virtualization is when the virtual manager is embedded in the circuits of the Hardware component instead of being called up from a third party software application. The virtual manager is called "Hypervisor".

In simple words,"It Provides Several Logical System on the Same Computer with Strong Isolation from Each Other."

Example:

1). KVM(Kernel Based Virtual Machine) for Linux System

2). VMWare(Virtual Machine Ware)

3). Xen

4). Cloud Computing

"The main advantage of Hard-ware based Virtual Machines is for effective utilization of hard-ware resources".

Software Based Virtual Machine

It is also known as Application Based or Process Based Virtual Machines. These Virtual Machines Acts as Runtime Engines to run a Particular Programming Language Applications.

Examples:

1). JVM acts as runtime engine to run Java Applications.

2). PVM(Parrot Virtual Machine) acts as runtime engine to run scripting language "PERL"

3). CLR(Common Language Runtime) acts as runtime engine to run .NET based Applications
