Method Overriding In Java

Declaring the method in the subClass which is already present in the parent class and changing it's implementation in it is known as method overriding.

The benefit of overriding is ability to define the behaviour according to the subclass type. It simply means the subcalss can implement a parent class method based on it's requirement.

A simple example of the method overriding is shown below:

class Animal{
public void move(){
System.out.println("Animals can move");
}
}

class Dog extends Animal{
public void move(){
System.out.println("Dog can walk and run");
}
}

public class Test{
public static void main(String[] args){
System.out.println("Main Method Started");
Animal a = new Animal(); //Animal reference pointing to Animal object
Animal b = new Dog(); //Animal reference pointing to Dog object

a.move(); //runs the method of Animal Class
b.move(); //runs the method of Dog Class
System.out.println("Main Method Ended");
}
}

In the above example, you can see that the even though b is a type of Animal it runs the move method in the Dog class. The reason for this is: In compile time, the check is made on the reference type. However, in the runtime, JVM figures out the object type and would run the method that belongs to that particular object.

Consider the following example:

Sample Code

class Animal{

public void move(){
 System.out.println("Animal can move");
 }
 }

class Dog extends Animal{

public void move(){
 System.out.println("Animal can move");

 System.out.println("Dog can bark");

public class TestDog{

public static void main(String[] args){
 System.out.println("Main Method Started");
 Animal a = new Animal(); // Animal reference variable pointing to Animal object
 Animal b = new Dog(); //Animal reference varialve pointing to Dog object

 a.move(); //runs the method in Animal class
 b.move(); //runs the method in the Dog class

}

The above example throws compile time error since reference variable "b" doesn't have reference to the bark() method of the Dog object.

Advantage of method overriding

The main advantage of the method overriding is that the subclass can give it's own implementation based on it's own specification without even modifying it's parent's class method implementation.

Rules for method overriding in Java

1). Argument List: The number and types of argument to the overriding method should be same as that of the parent's method implementation.

2). Access Modifier: The access modifier of the overriding method should be same and it should not be more restrictive than the overriden method of parent class. For e.g. if the access modifier of the base class is public than the access modifier of the subclass cannot be private,protected and default which is more restrictive than public access modifier.

The example where access modifier of the base overriding method is more restrictive(protected) than the overriden method(public) is shown below:

Sample Code

class BaseClass{
public void disp(){
 System.out.println("Inside Parent's class disp() method");
 System.out.println("Inside Parent's class disp() method");
 }
}

class ChildClass extends BaseClass{
protected void disp(){
System.out.println("Inside Base class disp() method");
 }
}

class TestClass{
public static void main(String[] args){
System.out.println("Main Method Started");
ChildClass obj = new ChildClass();
System.out.println("Main Method Ended");
}
}

3). Instance method can only be overriden if they are inherited by the subClass.

4). A method declared final cannot be overriden.

5). A method declared static cannot be overriden but can be re-declared.

6). If a method cannot be inherited, it cannot be overriden.

7). A subclass within the same package as the instance's superclass can override any superclass method that is not declared private or final.

8). A subclass in a different package can only override the non-final methods declared public or protected.

9). An overriding method can throw any uncheck exceptions, regardless of whether the overridden method throws exceptions or not. However the overriding method should not throw checked exceptions that are new or broader than the ones declared by the overridden method. The overriding method can throw narrower or fewer exceptions than the overridden method.

10). A constructor cannot be over iden.

Using the super keyword

When invoking a superclass version of an overidden method the super keyword is used.

Sample Code

class Animal{

public void move(){
 System.out.println("Animals can move");
 }
 }

class Dog extends Animal{

public void move(){
 super.move(); // invokes the super class method
 System.out.println("Dogs can walk and run");
 }
 }

public class TestDog{

public static void main(String args[]){

 Animal b = new Dog(); // Animal reference but Dog object
 b.move(); //Runs the method in Dog class

 }
 }

Output of the above example

Animal can move
Dogs can walk and run

Consider a scenario, Car is the parent class that provides functionality to get the Top Speed of the Car. But, Top Speed varies from car to car so it's implementation will be overrided in the overriding method in the base class as shown below.

The source code for this implementation is shown below:

Source Code

int cc;

}
}

 return 363;
 }
 }

class Pagani extends Car{
public double getTopSpeed(){
 return 335;
 }
 }

class Porshe extends Car{

 return 310;
 }
 }

Difference between Method Overloading and Method Overriding

The difference between Method Overloading and Method Overriding are given below:

Method Loading Method Overriding
Methods within the class having same name but differ to each other by number and types of data types in knows as Method Overloading. Method Overriding is used to provide the specific implementation of the method that is already provided by it's super class.
Method Overloading occurs within the same class. Method Overriding occurs within two or more class that have IS-A(inheritance) relationship.
In case of method overloading, parameter must be different In case of method overriding, parameter must be same.
In java, method overloading can't be performed by changing return type of the method only. Return type can be same or different in method overloading. But you must have to change the parameter. Return type must be same or covariant in method overriding.
