Aggregation In Java(HAS-A)

HAS-A means an instance of one class "has-a" reference to an instance of the another class or the same class. Composition(HAS-A) simply means use the instance variables that are references to other objects.

Consider we are storing the information of the student, we may create the class Student like this:

Class Student{
 int roll;
 String name;
 Address address;
 }

In the above example Class Student has an entity reference of the class Address. Here class Address stores again it's own information like street,city,state,zip. Like shown below:

class Address{
String street;
 String state;
 String zip;
 String city;
 }
So we can say that Student HAS-A address.

Consider the example below::


Student.java

public class Address{
String street;
 String city;
 String state;
String zip;

 public Address(String street,String city,String state,String zip){
 this.street = street;
 this.city = city;
 this.state = state;
 this.zip = zip;
 }
 }
