Method Overloading

Method Overloading is a feature which allow a class to have two or more methods having the same name but differs with each other by the type and numbers of argument.

Arguments list could differ in:

1).Number of parameters.

2).Data type of parameters.

3).Sequence of Data type of parameters.

Method OverLoading is also knows as Static Polymorphism.

Point To Note:

1). Static Polymorphism is also known as compile time binding or early binding.

2). Static binding happens at compile time. Method overloading is an example of static binding where binding of method call to its definition happens at Compile time.


Method Overloading Example

Example 1-OverLoading : Different Number of parameter in Argument List

Sample Code

class DisplayOverloading{
public void disp(char c){
 System.out.println("char::"+c);
 }
 public void disp(char c,int num){
 System.out.println("char::"+c+" & int::"+num);
 }
 }

class Sample{
public static void main(String[] args){
 DisplayOverloading obj = new DisplayOverloading();
 obj.disp('a');
 obj.disp('a',12);
 }
 }

Output:

char::a
char::a & int:: 12

In this example, method disp() is overloaded based on the data type of arguments – Like example 1 here also, we have two definition of method disp(), one with char argument and another with int argument.

Example 2-OverLoading : Difference in Data Typeof parameter Arguments

Sample Code

class DisplayOverloading{
public void disp(char c){
 System.out.println("char::"+c);
 }
 public void disp(int num){
 System.out.println("int::"+num);
 }
 }

class Sample{
public static void main(String[] args){
 DisplayOverloading2 obj = new DisplayOverloading2();
 obj.disp('a');
 obj.disp(12);
 }
 }

Output:

char::a
int:: 12

In the above example- The method in the class is beem overloaded by the means of the number of argument been passed to the methods. We have two defintion of the method with the same name disp() nut both of them differ according to the number of arguments been passed to it. One of the method accepts only int whereas the other accept both the char and the int.

Example 3-OverLoading : Sequence of Data Type of Arguments.

Sample Code

class DisplayOverloading3{
public void disp(char c , int num){
 System.out.println("I’m the first definition of method disp");
 }
 public void disp(int num, char c ){
 System.out.println("I’m the second definition of method disp");
 }
 }

class Sample3{
public static void main(String args[]){
 DisplayOverloading3 obj = new DisplayOverloading3();
 obj.disp('x', 51 );
 obj.disp(52, 'y');
 }
 }

Output:

I’m the first definition of method disp
I’m the second definition of method disp

Here method disp() is overloaded based on sequence of data type of arguments – Both the methods have different sequence of data type in argument list. First method is having argument list as (char, int) and second is having (int, char). Since the sequence is different, the method can be overloaded without any issues.


Lets see some Valid/Invalid case of Method OverLoading

Case 1:

Sample Code

int mymethod(int a, int b, float c)
int mymethod(int var1, int var2, float var3)

Result: Compile time error. This because the two method have same number and same sequence of data types in it's argument list.

Case 2:

Sample Code

int mymethod(int a, int b)
int mymethod(float var1, float var2)

Result: Perfectly fine. Valid case for overloading. Here data types of arguments are different.

Case 3:

Sample Code

int mymethod(int a, int b)
int mymethod(int num)

Result: Perfectly fine. Valid case for overloading. Here number of arguments are different.

Case 4:

Sample Code

int mymethod(int a, int b)
float mymethod(int var1, int var2)

Result: Compile time error. Argument lists are exactly same. Even though return type of methods are different, it is not a valid case. Since return type of method doesn’t matter while overloading a method.

Guess the answer before checking it out at the end of the program:

Question 1 – return type, method name and argument list same.

Sample Code

class Demo{
public int myMethod(int num1, int num2){
System.out.println("First myMethod of class Demo");
 return num1+num2;
 }
 public int myMethod(int var1, int var2){
 System.out.println("Second myMethod of class Demo");
 return var1-var2;
 }
 }
class Sample4{
public static void main(String args[]){
 Demo obj1= new Demo();
 obj1.myMethod(10,10);
 obj1.myMethod(20,12);
