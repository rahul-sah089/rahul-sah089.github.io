Block In Java

Java provides two types of blocks:

1). Static Block

2). Instance Block

1).Static Block

Static block is also called as Static Initilization Block. A static initialization blocks. is a normal block of code enclosed in braces, {} , and preceded by the keyword static keyword. Here is the example

Sample Code for Static Block

static{
//whatever code is needed for the initialization block goes here
 }

A class can have number of static intialization bocks, and they can appear anywhere in the class body. The runtime system guarantees that static intialization block runs in the order then appear in the source code. It is important to know the static initialization block will be executed when the JVm loads the class. JVM combines all the static block into a single block and then executes it.

Static block can be used to intialize the static variable of the class.

If the class is having both the static block and main method then static block will be executed prior to the main method invovation.

The example below shows how the static block is executed prior to the main method invocation:

Sample Code

class Demo1{
//1st static block
 static{
 System.out.println("Running 1st static block");
 }
 public static void main(String[] args){
 System.out.println("Main Method Started");
 System.out.println("Main Method Ended");
 }
 //2nd static block
 static{
 System.out.println("Running 2nd static block");
 }
 }

Output

Running 1st static block
 Running 2nd static block
 Main Method Started
 Main Method Ended

Another example of static block

Sample Code

public class StaticExample{
static {
 System.out.println("This is first static block");
 }

 public StaticExample(){
 System.out.println("This is constructor");
 }

 public static String staticString = "Static Variable";

 static {
 System.out.println("This is second static block and " + staticString);
 }

 public static void main(String[] args){
 StaticExample statEx = new StaticExample();
 StaticExample.staticMethod2();
 }

 static {
 staticMethod();
 System.out.println("This is third static block");
 }

 public static void staticMethod() {
 System.out.println("This is static method");
 }

 public static void staticMethod2() {
 System.out.println("This is static method2");
 }
 }

What will happen when you execute the above code? You will see below output.

Output

Running 1st static block
 This is second static block and Static Variable
 This is static method
 This is third static block
 This is constructor
 This is static method2

Now lets the output. First all static blocks are positioned in the code and they are executed when the class is loaded into JVM. Since the static method staticMethod() is called inside third static block, its executed before calling the main method. But the staticMethod2() static method is executed after the class is instantiated because it is being called after the instantiation of the class.

2).Instance Block

Instance Initilization Block is been used to intialize the instance data member of a class. It will run each time when the object is created.

Instance member is used to initialize instance member of the class.

Instance member executuion depends on the how many times the object is been created. It get executed each time the object is created.

Below shows the example how the instance block is been executed:

Sample Code

public class InstanceBlock {
//1st instance block
 {
 System.out.println("runing 1st instance block");
 }
 //Main Method implementation
 public static void main(String[] args){
 System.out.println("Main Method Started");
 InstanceBlock obj1 = new InstanceBlock();
 System.out.println("-------------------------");
 InstanceBlock obj2 = new InstanceBlock();
 System.out.println("Main Method Ended");
 }
 //2nd instance block
 {
 System.out.println("running 2nd instance block");
 }
 }

Output

Main Method Started
runing 1st instance block
running 2nd instance block
-------------------------
runing 1st instance block
running 2nd instance block
Main Method Ended

Which is invoked first? Constructor or instance block?

The answer for this question is instance block.Lets have a look in the example below:

Sample Code

public class Invoked1st {

//Constructor method
Invoked1st(){
 System.out.println("Constructor invoked 1st");
 }
 //Instance Block
{
 System.out.println("Instance block invoked 1st");
 }
 public static void main(String[] args){
 System.out.println("Main Method Started");
 Invoked1st obj = new Invoked1st();
 System.out.println("Main Method Ended");
 }
 }

The output of the above example is as follows:

Output

Main Method Started
Instance block invoked 1st
Constructor invoked 1st
Main Method Ended

Which block will get executed first? Static Block or Instance Block?

Below show a example to proof which block get executed first:

public class Executed1st {

Executed1st(){
 System.out.println("Object Created");
 }

 static{
 System.out.println("Static bock gets executed");
 }

 {
 System.out.println("Instance block gets executed");
 }

 public static void main(String[] args){
 System.out.println("Main Method Started");
 Executed1st ref = new Executed1st();
 System.out.println("Main Method Ended");
 }
 }

The output of the above program is as follows:

Output

Static bock gets executed
Main Method Started
Instance block gets executed
Object Created
Main Method Ended

From the above program and it's output we can draw the following conclusion:

1). Static block is the block to get executed first.

2). Instance block gets executed once the object is created.

3). Instance block gets executed before the constructor method invocation.

So, let us summaries the blocks with the difference between the static block and instance block.

Static Block Instance Block
It gets executed at the time of class loading by class loader. It gets executed at the time of object creation.
It is used to initialize the static member of the class It is used to initialize the instance member of the class
Static block is executed only once. Instance block gets executed every time the object is created.
In a class static block gets executed first. In a class instance block gets executed after the static block execution.
