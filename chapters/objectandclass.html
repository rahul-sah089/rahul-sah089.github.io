Object and Classes in OOPs

In this section we will learn about Object and Classes. Object is the building block in Object Oriented Programming Language. It is important to know the difference between the Classes and Object.

With the basics knowledge of the Java programming Language, you can learn to write your own classes. In this section we will learn how to define our own classes, including member variables, methods ad constructors.

Object is the physical as well as logical entity whereas class is the logical entity only.


What is an Object?

An entity which is having state and behaviour is knows as Object like mobile phone, watch, laptop, pen, etc. It is the physical as well as logical entity. Physical means it can be exist in the scope of the program or in the memory location.An object consist of the these characteristics:

1). State: It represent the data of an object. If we consider a dog, then it states are : name,breed etc

2). Behaviour: It represent the functionality of and object. If we consider a dog, then ite behaviour are : barking,wagging and running

3). Identity:Each object has it's own identity and the every object differs from one another. Object identity is implemented via a unique ID. THis value is not visible to the user but JVM uses it internally to distinguish an Object from another.

Similarly objects also have state and behaviour. A software object's state is stored in fields and the behaviour is shown via method.

It should be noted that Class is the template/blueprint of object using which the object is been created.So object is an instance of the class.

What is Classes In Java?

A class is a blueprint using which the object is been created.

A sample of class is given below:

public class Dog{
String breed;
 int age;
 String color;

 void barking(){
 }

 void hungry(){
 }

 void sleeping(){
 }

 }

A class in Java can contain the following:

1).Data Member

2).Method

3).Constructor

4).Block

5).Class And Interface

A class can contain any of the following variable types:

1). Local Variable: Variable defined inside the constructor, method or blocka are called Local Variables. The variables will be declared and initialized within the method and it scope remains within it. Once the method returns the scope of the local variable is lost.

2). Instance Variables:Instance variable are the variable declared inside the class but outside the method. This variable is the entity of class and new copy of them is created at every object creation. Instance Variable are accessable within the method, constructor or blocks of particular class.

3). Class Variables:It is the variable defined inside the class but outside the method with the keyword static with it. The static variable concept it that there is exactly one copy of this variable existing in the memory and it is shared among all the instance of the class.

Simple Example of Objects and Class

Sample Code

class Student{
int id; //instance variable
 String name;//instance variable

 public static void main(String[] args){
 Student stud = new Student();//creating an object of Student
 System.out.println(stud.id);
 System.out.println(stud.name);
 }
 }

Output of Code

0
null

Explanation of the Code

1).The class name is defined by giving it's name after the class keyword.

2).Declare the variable inside the class. Here the variable are Instance Variable since they declared inside class but outside the method declaration and without the keyword static in it.

3).The main method declaration is been given. main Method is the 1st method from where the Java Program execution starts. It is been declared as static since it JVM don't have to create the object of the class explicitly.

4).Inside the main method declaration we are creating the object of type Student. It is created using the keyword new operator which will allocates object memory at runtime. The object of the Student class is referred using the reference variable stud. This hold the address of the memory location where the Student object is stored.

5). The instance variable of the object can be accessed using the reference variable of it i.e; reference variable name followed by "."(dot) followed by instance variable name.

OOPs programming model programs are organized around the object and data rather than actions and logic. Objects in OOPs represent some concepts or things like any other Objects in the real Objects and have certain behaviour, properties, type, and identity.


The figure shows an Object Dog whose states are Name,Color,Breed, Hungry and the behaviours are Barking,Fetching, Wagging tail




The OOPs language simplifies the software development and maintenance by providing some concepts like::

1).Object

2).Class

3).Inheritance

4).Polymorphism

5).Abstraction

6).Encapsulation

Object

Objects are the key to understanding object-oriented programming. If we look aroung we will find many examples of objects. Like your chair, your laptop, your mobile,your pen etc

Real world objects have two characteristics. They all have state and behaviour.For example every person have a state(name,age,height,weight) and behaviour(walking,sitting,eating,sleeping etc). This state can vary from object to object. Identifying the state and behavior for real-world objects is a great way to begin thinking in terms of object-oriented programming.

Classes

Classes serves as the blue print for the object. It is the logical entity which doesn't exist in memory area. Objects are the instances of the Class.

Inheritance

The concept in which an object accquires all the states and behaviour(properties) of parent object i.e; knows as inheritance.It provides code reusability. It prevents the programmer from unnecessary work and it is the inheritance concept that does this job. It is also used to achieve runtime polymorphism which we be discussed in further details in further sections.

Polymorphism

Polymorphism allows an entity to take a variety of representation.Polymorphism means the ability to request that the same Operations be performed by a wide range of different types of things. Effectively, this means that you can ask many different objects to perform the same action.

In Java, polymorphism can be achieved using method overloading and method overriding.

The example of the polymorphism can be speak() function. Different animals can speak in different way. cat speaks meaw, dog barks woof etc.

Abstraction

Abstraction is the process of hiding the Class Implementation by the usage. In Abstraction the implementation of the object is never exposed and not shown to the usage part of the code. Only the object functionality is been shown.

In Simple Words "Abstraction is hiding the internal details and showing the functionality".For example : Car, we don't know how the car works and functions internally.

Abstraction can be achieved by fulfilling the following process:

1). Generalize the behaviour of the sub-classes in the interface.

2). Implement the behaviour in the subClasses according to it's specification..

Abstraction will be discussed in much detail in the coming tutorial sections.

Encapsulation

It is the mechanism that binds code and data together and keep them safe and away from outside interference and misuse. For example : A capsule, it is wrapped with different medicines.

A Java Class is an example of encapsulation. A Java Bean Class is a fully encapsulated class because all the data members are private and to access it data member we need setter and getter method.

What is the relation between Classes And Object?

They look very much similar but not same. Class is a definition, while object is the instance of the class created. Class is blue print while objects existing in the real world.Example we have class CAR which has attributes and methods like Speed, Brakes, Type of Car etc.Class CAR is just a prototype, now we can create real time objects which can be used to provide functionality. Example we can create a Maruti car object with 100 km speed and urgent brakes.

Advantage of OOPs over Procedure-oriented programming language

1)OOPs makes development and maintenance easier where as in Procedure-oriented programming language it is not easy to manage if code grows as project size grows.

2)OOPs provides data hiding whereas in Procedure-oriented prgramming language a global data can be accessed from anywhere.

3)OOPs provides ability to simulate real-world event much more effectively. We can provide the solution of real word problem if we are using the Object-Oriented Programming language.
